CC6.1 — Logical and Physical Access Controls

PARTIAL
The entity implements logical access security software, infrastructure, and architectures over protected information assets.

Control Implementation

The organization enforces logical access controls through Kubernetes Role-Based Access Control (RBAC), which provides granular permission management across all cluster resources. The RBAC configuration comprises 47 ClusterRoles and 23 ClusterRoleBindings. Access to the cluster-admin role, which grants unrestricted administrative privileges, is restricted to 2 authorized bindings: system:masters (Group) and break-glass-admin (User).

Service account credential exposure is controlled across all 12 audited namespaces. Automatic mounting of service account tokens is disabled, preventing unintended credential availability to workloads that do not require Kubernetes API access.

Findings requiring attention:
  • CIS-5.1.3 and RBAC-3: 2 ClusterRoles use wildcard resources: legacy-admin, dev-full-access (2 independent checks confirm this finding)

Assessment: PARTIAL — 16 checks passing and 2 checks failing.
